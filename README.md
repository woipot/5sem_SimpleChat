# 5sem_SimpleChat
GUI chat on sockets (python - server : qt c++ -  gui client)

Чат. Реализовать комплекс программ, состоящий из
консольного приложения на Python (сервер) и оконного приложения на Qt C++ (клиент), для
создания простейшего текстового чата. При реализации обоих приложений запрещается
использование сторонних библиотек. Обмен данными между клиентом и сервером должен
быть реализован через сокеты.
 Сервер должен обрабатывать одновременно несколько клиентов. При входе в чат
пользователя через клиентское приложение, сервер должен запрашивать логин
пользователя, состоящий из 4-20 символов, являющихся символами цифр, символами
букв латинского алфавита в любом регистре, символами ‘_’ и ‘-‘, при этом логин не должен
начинаться с символа цифры или символа ‘-’; он также не должен совпадать с логином
пользователя, уже участвующего в чате. При получении некорректного логина, сервер
должен сообщить об этом клиенту. О подключении к чату нового пользователя или об 
отключении участвующего в чате пользователя должны узнать все остальные
пользователи. Сервер должен быть способен обрабатывать 10 и более клиентов
одновременно. Версия языка Python должна быть 3.6.5 и выше.
 Клиентское приложение должно содержать меню с пунктами “Disconnect” (отключение
от сервера чата, должен быть неактивен при отсутствии подключения) и “Connect”
(аналогично “Disconnect”, неактивен при наличии подключения); по нажатию на
последний необходимо предоставить пользователю возможность ввести через
модальный диалог логин (валидация логина пользователя должна производиться на
сервере, при некорректно введённом логине – необходимо обработать ответ сервера и
сообщить пользователю об ошибке при помощи вызова QMessageBox::critical), а также
адрес сервера (в формате адреса IPv4, с валидацией на клиенте) и порта (валидация на
клиенте). После успешного подключения, пользователь может отправлять в чат простые
текстовые сообщения (сообщения вводятся при помощи элемента управления QLineEdit,
отправляются при помощи элемента управления QButton), которые должны быть
обработаны сервером и отправлены остальным подключённым пользователем.
Необходимо корректно обработать отключение клиента от сервера чата (при отключении
от сервера через пункт меню “Disconnect”, либо через попытку закрытия формы
клиентского приложения, необходимо запросить подтверждение пользователя). Во
время общения в чате и после отключения клиента от сервера чата, должна иметься
возможность сохранения истории сообщений в текстовый файл (имя файла указывается
при помощи QFileDialog). При повторном подключении к чату необходимо локально
очистить историю сообщений.
Текстовые сообщения чата, отображаемые в клиентском приложении, должны иметь
формат
[<datetime>]{<login>}: <message>
, где <datetime> – дата и время отправки сообщения, <login> – логин пользователя,
отправившего сообщение, <message> – содержимое сообщения.
Сообщение подключения пользователя к чату должно иметь формат
[<datetime>] User <login> joined the chat.
, где <datetime> - дата и время подключения пользователя к чату, <login> - логин
подключённого пользователя.
Сообщение отключения пользователя от чата должно иметь формат
[<datetime>] User <login> left the chat.
, где <datetime> - дата и время отключения пользователя от чата, <login> - логин
отключённого пользователя.
Вся валидация должна быть реализована посредством применения регулярных
выражений.
